WealthBridge Financial Data Platform

Enterprise Real-time Data Ingestion, Quality Assurance & Data Lakehouse Architecture

Architectural Deck
Built with AWS Serverless, PySpark, & Apache Iceberg
The Business Challenge
Use Case & Problem Statement
  • Unstructured Data Streams: Legacy upstream pipelines emit disjointed batches of transaction and position records without transactional guarantees.
  • Out-of-Order Delivery: Sibling data uploads (e.g. Transactions vs Positions) arrive asynchronously, causing processing faults.
  • Data Quality Risk: Invalid currencies, duplicated ledger IDs, and negative pricing corrupt downstream calculations.

Goal Statement

Build a resilient, cost-efficient, zero-maintenance pipeline that validates data quality dynamically, quarantines bad records, and writes transactional outputs to a curated lakehouse.

System Design
Project Solution Overview

Event-Driven Push Architecture

No cron scheduling. Files are processed instantly upon arrival via S3 events, Preprocessor Lambda gates, and AWS Glue ETL compute.

  • Deduplicated Ingestion: Strict manifest validation ensures that already processed files are blocked instantly.
  • Dynamic Quality Registry: Relational SQL rule database loads validation constraints dynamically at runtime (no code re-deploys).
  • Open Lakehouse Tables: Written to S3 in Apache Iceberg format for ACID transactions and partition evolution.
Data Flow
Chronological Operational Workflow
  • 1. Drop: CSV files are uploaded into the S3 Landing Bucket.
  • 2. Match: Preprocessor Lambda fuzzy matches names to TRANSACTION or POSITION.
  • 3. Record: Key metadata is inserted into the file_manifest table.
  • 4. Evaluate: Lambda checks if both matching files of a file_id are registered.
  • 5. Trigger: If complete, status is set to TRIGGERED and the AWS Glue job runs.
  • 6. ETL & Write: Glue loads active rules, cleanses data, and appends to Iceberg tables.
Compute Optimizations
Gatekeeper Orchestrator (AWS Lambda)

Instead of triggering expensive Spark clusters directly from S3 events, the pipeline utilizes a lightweight **AWS Lambda** function. It queries PostgreSQL to verify pair completeness before starting Glue.

Business Advantages

  • 90% cost savings by preventing Glue from starting with partial files.
  • Stateful hold: safely caches out-of-order uploads.
  • Zero server footprint: runs serverless, billing only for execution milliseconds.
Data Governance
Configuration-Driven DQ Engine
  • No Hardcoding: Business constraints are cataloged as standard SQL statements inside dq_rule_registry.
  • Dynamic Compilation: The Engine compiles SQL registry checks into Spark SQL expr() boolean filters at run time.
  • Failure Accumulation: Stores all failed rule codes and error descriptions in a vector rather than failing on the first error.

Supported Rule Types

NOT_NULL, UNIQUE (within-batch duplicates rejected), ALLOWED_VALUES (allow-list validation), and EXPRESSION (Spark Boolean SQL check).

Data Lakehouse
Target Storage (Apache Iceberg)

Why Apache Iceberg?

Provides transactional consistency on S3 data lakes. Good records flow to _good curated tables; bad records flow to _bad quarantine tables.

  • ACID Transactions: Guarantees atomic commits and rollbacks. Re-running a job never creates duplicate lines.
  • Schema Evolution: Columns can be added or updated dynamically over time without breaking historical logs.
  • Hidden Partitioning: Automatically partitions datasets by batch_id for sub-second Athena query lookups.
Reporting & Audit
Operational Correction Assistant
  • Consolidated Dashboard: Generates an interactive dark-mode HTML report summarizing metrics, stats, and rule failure frequencies.
  • AI Analyst Summary: Translates stats into plain-english analysis (e.g. flagging specific columns causing the most failures).
  • Actionable Repair Guidelines: Maps failed rule IDs directly to correction instructions for operations teams.

Latest Report Path

Saved at reports/latest_report.html and archived by run ID under reports/ for easy browser preview.

Future Scope
Production Readiness & Improvements
  • Secrets Manager Integration: Dynamically retrieve RDS passwords via AWS Secrets Manager instead of environment variables.
  • CI/CD Pipelines: Automate test execution (pytest) and SAM deployments via GitHub Actions.
  • Real-time Alerting: Integrate AWS SNS to send emails/Slack alerts on partial or failed runs.
  • Monitoring Dashboards: Connect Datadog or CloudWatch to track Glue run times and SLA limits.
  • Data Warehousing: Expose curated Iceberg tables directly in Redshift Spectrum for BI reporting.
  • DLQ Pattern: Move rejected files to an S3 Dead Letter Queue (DLQ) for auto-notification.
Conclusion
Key Project Outcomes

Measurable Impact

Delivers a secure, transactional ledger intake pipeline that runs at minimum AWS compute cost and scales horizontally on demand.

  • 99.9% cost reduction for scheduling, relying on events instead of idle EC2 nodes.
  • Zero redeployment friction for business operations changes.
  • Fast resolution time: clear guidelines reduce debugging from hours to minutes.
